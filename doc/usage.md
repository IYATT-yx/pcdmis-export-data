# 使用说明

## 在测量程序中添加本工具

pcdmis-export-data（下称工具）是以 BASIC 脚本和外部命令的方式进行工作，在需要自动导出数据的测量程序中添加调用工具的命令即可，支持图形化操作。  
下载工具压缩包后，解压到一个固定目录，防止意外删除，双击运行 `pcdmis-export-data_win_amd64`  
![alt text](img/image4.png)  
打开 PC-DMIS，并打开需要添加自动导出数据的测量程序，在工具中点击`添加本工具`，会在测量程序的末尾进行添加  
如果勾选了「保存测量程序副本」，则会额外存储测量程序的副本（有采集数据但是没有评价的，后期还可以补救）  
如果勾选了「保存PDF报告」，则会自动保存PDF报告

### 指定输出目录

可通过 `-o` / `--output` 参数在命令行指定数据输出目录，UI 界面也提供了输出目录选择行（含浏览按钮），留空则使用默认 `data` 目录。

## 用于追溯产品的序列号

工具会优先使用测量程序中赋值的`SN`变量用作序列号，如果没有赋值`SN`变量，则使用测量程序头部的`序列号：`填写的内容  
![alt text](img/image8.png)

考虑到检测员在运行测量程序前可能忘记填写序列号，因此设计了`SN`变量用于替代默认的序列号，可以在测量程序中添加一个注释用于提示检测员输入序列号，然后把注释读取到的检测员输入的内容赋值给`SN`变量。工具在检测到`SN`变量有输入内容后，会同步把`SN`值赋值给测量程序的序列号，由于默认 PDF 报告模板中序列号的字段也是用的测量程序的序列号值，这样相当于同步了 PDF 报告的序列号。

把光标点到要添加输入注释的位置，点击「添加序列号输入」，会添加到光标位置的命令的后面。  
我建议是添加到手动找正建坐标系后，这个时候提示录入，接着就开始自动检测直到完成。这样输入的序列号可以显示在第一个评价项之前（有时候追溯码过长，表头显示追溯码不全，这里的注释输入默认会显示到报告上）。  
![alt text](img/image9.png)  
![alt text](img/image10.png)

如果是使用扫码枪录入产品序列号，建议勾选「强制切换英语键盘输入」。扫码枪是被电脑当作键盘的，如果在扫码的时候是中文输入法，则可能导致录入的内容和二维码原始内容不一致，比如把输入法的候选内容键入了，或者下划线(`Shift` + `-`)变成了破折号。这都是我试验踩坑后，才增加的功能。

每次测量的数据以行呈现。
